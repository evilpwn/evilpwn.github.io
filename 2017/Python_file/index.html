文件

  • 存储在外部介质上的数据或信息集合
  • 有序的数据序列

常用的编码

ASCII码

  • 7位二进制位编码
  • 表示128个字符
  • 最通用的单字节编码

Unicode

  • 跨语言、跨平台
  • 统一且唯一的二进制编码
  • 每个字符两个字节长（也称宽字节集（Widechars））
  • 65535个字符的编码空间

UTF-8

  • 可变长度的Unicode

Unicode与UTF-8编码字节范围对应关系：

Unicode符号范围（十六进制） UTF-8编码方式（二进制）
0000 0000-0000 007F 0xxxxxxx
0000 0080-0000 07FF 110xxxxx 10xxxxx
0000 0800-0000 FFFF 1110xxxx 10xxxxx 10xxxxx
0001 0000-0010 FFFF 11110xxx 10xxxxx 10xxxxx 10xxxxx

GBK编码

  • 《汉字内码扩展规范》
  • 双字节编码

Python中，字符串类型未编码
编码：encode(“utf-8”)
解码：decode(“utf-8”)

文件数据

  • 文本文件
  • 二进制文件

文本文件

以ASCII码方式存储

二进制文件

照片、音乐、视频、计算机程序等

优点：1)更加节省空间 2)采用二进制无格式存储 3)表示更为精确

对比

  • 文本文件是基于字符定长的ASIIC。
  • 二进制文件编码是变长的，灵活利用率高。
  • 不同二进制文件解码方式不同。

打开文件

打开模式
r 只读
w 只写
a 附加文件末尾
rb 只读二进制
wb 只写二进制
ab 附加到二进制文件末尾
r+ 读写，写时会附加在文件末尾

若文件不存在，w会创建，其他的（包括rw，r+）都会报错。

文件读取

  • read([size])
    从文件读取指定的字符数，如果未给定或为负则读取所有。
  • readline([size])
    读取整行，包括 “\n” 字符，若size小于1行的字符，则读到size处。
  • readlines()
    返回值为整个文件内容的列表，每项是以换行符为结尾的一行字符串。

文件写入

  • write(str)
    将字符串写入文件，没有返回值。
  • writelines(sequence)
    向文件写入一个序列字符串列表，如果需要换行则要自己加入每行的换行符。

其他操作

seek()

seek() 方法用于移动文件读取指针到指定位置。

file.seek(offset[, whence])
offset：开始的偏移量，也就是代表需要移动偏移的字节数
whence：可选，默认值为 0。给offset参数一个定义，表示要从哪个位置开始偏移；0代表从文件开头开始算起，1代表从当前位置开始算起，2代表从文件末尾算起。

name

file.name 返回文件对象的文件名。
